将私有仓库的构建文件发布到另一个公开仓库 ​

需求 ​

仓库A是不开源的，是我的项目源码，但是我想要将仓库A的build文件，上传到另外一个GitHub公开仓库B的main分支，同时仓库B的main分支已经设置为pages，这样就能方便别人访问和使用

仓库A的actions文件

yml
# name 可以自定义
name: Deploy GitHub Pages

# 触发条件：在 push 到 main/master 分支后，新的 Github 项目 应该都是 main，而之前的项目一般都是 master
on:
  push:
    branches:
      - main

# 任务
jobs:
  build-and-deploy:
    # 服务器环境：最新版 Ubuntu
    runs-on: ubuntu-latest
    steps:
      # 拉取代码
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
      
      # 生成静态文件
      - name: Build
        run: |
          npm i pnpm -g
          pnpm install
          pnpm run build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v4
        with: 
          personal_token: ${{ secrets.PERSONAL_TOKEN }} # 令牌
          publish_dir: docs/other/build # 构建后的文件目录
          external_repository: nsnans/vitepress-blog # 仓库B，格式为：user/repo
          publish_branch: page # 仓库B的分支

参考

  • https://developer.aliyun.com/article/1425333

部署之后,github page的custom domian自定义域名失效

主文件夹下新建CNAME文件，比如public里面，里面填入你的域名，并保存，github page就不会出现域名重置了
